KEY DESIGN DECISIONS
--------------------------------------------------------------------------------
- Output format: strict JSON only
- Setting 1 (Textual CoT):    {"reasoning": ..., "box": null, "answer": ...}
- Setting 2 (Visual CoT):     {"reasoning": ..., "box": [...], "answer": ...}
- Setting 3 pass 1:           {"reasoning": ..., "box": [...], "answer": ...}
- Setting 3 pass 2:           {"verification_reasoning": ..., "box": [...], "answer": ...}
- Box: normalized [xmin, ymin, xmax, ymax] in [0,1]
- Target region: union of obj1_bbox + obj2_bbox from COCO (normalized)
- RSA = 1 if IoU(pred_box, target_box) >= 0.5
- Generation: greedy (do_sample=False) for reproducibility
- Dev subset: 20 examples from VSR validation split (DEV_SUBSET_SIZE in load_vsr.py)

--------------------------------------------------------------------------------
SETTING 3 — VISUAL CoT + VERIFICATION (two-pass design)
--------------------------------------------------------------------------------
Pass 1:
  - prompt: visual_cot_pass1.txt  →  {caption}
  - model outputs: reasoning, box, answer
  - parsed as: pass1_reasoning, pass1_box, pass1_answer

Pass 2:
  - prompt: visual_cot_verification_pass2.txt  →  {initial_box}, {caption}
  - model sees ONLY its own pass-1 box (not its reasoning or answer)
  - asks: is this box good? revise if needed. give final answer.
  - model outputs: verification_reasoning, box, answer
  - parsed as: pass2_verification_reasoning, pass2_box, pass2_answer

Both passes run in a single job (model loaded once).
Script: src/run_visual_cot_verification.py
SLURM:  scripts/run_visual_cot_verification.sh  (3h limit, ~55 min actual)

Result schema per example:
  example_id, caption, relation, subj, obj, ground_truth_answer, target_box
  pass1_raw_output, pass1_parsed_json, pass1_reasoning,
  pass1_box, pass1_box_valid, pass1_box_invalid_reason,
  pass1_answer, pass1_answer_correct, pass1_iou, pass1_rsa
  pass2_raw_output, pass2_parsed_json, pass2_verification_reasoning,
  pass2_box, pass2_box_valid, pass2_box_invalid_reason,
  pass2_answer, pass2_answer_correct, pass2_iou, pass2_rsa
  box_revised, answer_flipped, recovered

Summary printed:
  pass1 FAA, pass2 FAA, pass1 RSA, pass2 RSA,
  box revision rate, answer flip rate, recovery rate

MULTISTAGE DIAGNOSIS AND FIX (2026-04-26)
--------------------------------------------------------------------------------

PROBLEM:
  Before fix — clean condition:
    Initial FAA:  87.4% (111/127)
    Final FAA:    65.4% (83/127)   <- 22 point drop from correction
    Triggered:    84.3% (107/127)  <- fires on almost everything
    Recovered:     4.7% (6 examples flipped wrong->right)
    Degraded:     26.8% (34 examples flipped right->wrong)
    Mean IoU:      0.108  max=0.562

ROOT CAUSE 1 — BQS trigger miscalibrated:
  triggered = not box_valid OR bqs < 0.6
  BQS overlap_score (weight=0.50) = IoU(predicted_box, COCO_ground_truth_box)
  Mean IoU=0.108 because model localizes what it thinks is relevant, not COCO boxes.
  So correction fires on ~84% of examples regardless of actual box quality.

ROOT CAUSE 2 — Crop destroys spatial context:
  Stage 5 cropped to the predicted box, then asked the model on the crop.
  VSR questions are relational ("is A left of B?") — cropping to one object
  removes both objects and the space between them. Model can't answer spatially.

FIX #1 — trigger condition (eval_runner.py + run_corrupted.py):
  Before: triggered = not br1["valid"] or bqs_prov["bqs"] < BQS_THRESHOLD
  After:  triggered = not br1["valid"]
  Correction now fires only on format failure (~5-10% of examples).

FIX #2 — Stage 5 full-image re-ask (eval_runner.py + run_corrupted.py):
  Before: crop_img = safe_crop(img, revised_box); model on cropped image
  After:  model on full image with revised_box coordinates in the prompt text
  Prompt: prompts/multi_stage_reask_answer.txt
          placeholders: {revised_box}, {caption}
          JSON keys: answer_reasoning, answer
  Result dict fields: crop_* -> reask_* (reask_raw_output, reask_parsed_json, reask_reasoning)
  safe_crop import removed from both files.

STATUS (as of 2026-04-26):
  Fix verified — triggered=0/127 on clean, final_FAA=initial_FAA=87.4%.
  All 10 multistage conditions re-run and aggregated.

Verification dynamics (clean condition):
  Method         BRR     AFR   Recov
  multistage   100.0%    0.0%    0.0%   (correction never fires — triggered=0)
  verification  20.5%   14.2%   28.0%

KEY FINDINGS:
1. Multistage has highest FAA but only because correction never fires (format failure
   rate ~0%). It is functionally equivalent to visual CoT pass 1. Correction would
   need a smarter trigger to add actual value.

2. Visual CoT is the most robust method — negative PDR for blur/noise means it
   actually improves slightly on corrupted images vs clean. Bounding box grounding
   helps on blurred/noisy inputs.

3. Verification underperforms clean FAA but has 28% recovery rate — it helps some
   wrong examples but also flips correct ones (14.2% flip rate). Net negative on clean.

4. All methods degrade ~3% under rotation (geometry change). Rotation PDR is the
   most consistent robustness signal.
